Клиент-серверная архитектура

Реализует сетевое взаимодействие двух узлов, в котором узлы играют роли
клиента и сервера.

Сеть (ЛВС - локальная вычислительная сеть, LAN - Local Area Network) - 
 совокупность сетевых узлов и каналов связи между ними
Сетевой узел - активный (вычисляющий) элемент сети с назначенным
 сетевым именем или сетевым адресом (ПК, маршрутизатор, принтер, сканнер, телефон, ...).
Канал связи - способ обмена данными между узлами.

Топология сети - схема (устройство) сети, организация каналов связи
 = кольцо - коммуникационный канал образует замкнутую цепь, все узлы подключаются
            к этому каналу
 = звезда - некоторые сетевые узлы являются концентраторами, подключающими
            подсеть других узлов (современная топология)
 = шина   - похожа на кольцо, но канал не замкнут

Сетевой протокол - набор правил, по которым формируются сообщения между узлами.
 = TCP - P2P протокол, сообщение имеет адресата
 = UDP - широковещательный протокол, данные получают все узлы

Сетевая технология - совокупная характеристика: топология, протокол, другие хар-ки
 свойственные данной сети. Современная-популярная технология - Ethernet (TCP/IP, звезда)

IP - сетевой адрес, число присваиваемое сетевому узлу
 IPv4 - число 32 бита, разделенное для простоты на 4 байта: 127.0.0.1
 IPv6 - 128 бит, разделяемые на группы через ":" 
        2001:0db8:85a3:0000:0000:8a2e:0370:7334,
        ::1

Порт - число, присваиваемое процессу (работающей программе) на сетевом узле - способ
        разделения разных программ на одном узле. Процесс "слушает" (listen) порт

Endpoint - идентификатор процесса - адрес узла + порт

Кодировка канала - способ преобразования исходных данных в бинарные, передаваемые
  по каналу

Прикладной протокол - набор правил, по которым анализуруются данные, переданные по сети
  например НТТР

----------------------------------------------------------------------------------------
Сервер - роль сетевого узла, при которой узел принимает запрос, обрабатывает и, обычно,
 возвращает ответ (Callee - тот, которого вызывают, кому звонят)
 Сервер (как процесс) постоянно активен, слушает порт в ожидании запроса от клиента

Клиент - роль, при которой узел инициирует "общение", посылая серверу запрос
 (Caller - звонящий, вызывающий)
 Клиент (как процесс) проявляет импульсную активность: подключение-обмен данными-отключение

После установки соединения роли "сравниваются" и каждый из участников эквивалентен

Сокет - программная абстракция, скрывающая под собой процедуру обмена данными между
 программами (процессами). С т.з. программы происходит запись в сокет и чтение из
 сокета, остальные детали процесса скрываются термином "сокет"
----------------------------------------------------------------------------------------
О UTF-8
Мультибайтная кодировка - количество байт, определяющее один символ, разное для разных
символов. Для символов с кодами меньше определенного (~220) берется сам код и его
ASCII символ.
!! Это можно считать рекомендацией при испытаниях выходить за ASCII - использовать кириллицу
Если первый байт больше, то и следующий байт включается в код символа
3A FD 2C 47
1  --2-- 3

Мультибайтность создает предпосылки для неправильного разделения строки, если делить
по байтам   3A FD / 2C 47  - неправильно, символ разорван
Поэтому для таких строк желательно сначала соединить все байт-фрагменты (chunk)
и только потом декодировать их в символы
