HTTP - HyperText Transfer Protocol - протокол передачи гипертекста
Исторически - для передачи HTML, в современности - обменный протокол
универсального характера.

HTTP/1.0
HTTP/1.1
HTTP/2
HTTP/3

Суть
Два участника обмена - клиент и сервер. Единица обмена - пакет.
Пакеты бывают двух видов: запрос (request) и ответ (response)
Любой пакет состоит из трех частей
- статус-строка - первая строка запроса
Запрос                       Ответ
GET /auth HTTP/1.1			 HTTP/1.1  200  OK

- заголовки - начиная со второй строки и до первой пустой строки,
  в одной строке - один заголовок по схеме   Ключ: значение; атрибуты
Content-Type: text/html; charset=utf-8
Connection: close

- тело пакета - произвольные данные после пустой строки и до конца пакета


Обмен данными по НТТР:
клиент формирует запрос: указывает
 - метод запроса (GET, POST, PUT, DELETE, ...) по умолчанию - GET
 - URL запроса - разбивается на 2 части host, path
        site.com/auth ->
    GET /auth
    Host: site.com
- при необходимости добавляются заголовки (ключ - значение)
- при необходимости заполняется тело

Запрос отправляется, ожидается ответ
 - статус-код: трехзначное число, в котором первая цифра характеризует состояние
     1 - переключение протокола
     2 - успех (200)
     3 - перенаправление (301)
     4 - ошибка запроса (404)
     5 - ошибка сервера (500)
 - заголовки ответа
 - тело ответа


В НТТР все переводы строк строго "\r\n"
